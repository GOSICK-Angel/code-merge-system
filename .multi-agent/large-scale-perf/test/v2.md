## 6. 覆盖率维持策略

### 6.1 目标

CLAUDE.md / facts.md K4：总覆盖率 ≥80%（`pyproject.toml --cov-fail-under=80`）；新增模块自身覆盖率不能拉低基线。

### 6.2 基线对比方法（P1-5 修订）

- **基线测定**：Phase 0 commit #1 前 Executor 必须执行 `pytest tests/unit/ --cov=src --cov-report=term --cov-report=json:.coverage-baseline.json -q`，记录 `total_pct` 入 `code/phase-0/v1.md` 的 "baseline" 段。
- **每 Phase 验证**：每个 Phase 末 commit 前用同一命令产出新 `coverage.json`，对比：
  - 门槛 1：`total_pct >= 80.0`（`--cov-fail-under=80`，pyproject.toml 强制）。
  - 门槛 2：`new_total_pct >= baseline_total_pct - 0.5`（**允许 0.5 个百分点漂移容差**，对应 ±数十 LOC 随机波动）。
- **per-Phase 局部覆盖率（仅参考，不卡死）**：
  - Phase 0 新增模块（RunBudgetExceeded、AgentContract.version、7 yaml）属纯结构，自带 100% 覆盖。
  - Phase 1 `_aggregate_chunked_analyses` reducer：目标 ≥85%（U-P1.2 / U-P1.3 / U-P1.4 / U-P1.5 / U-P1.6 / U-P1.12 6 条分支基本全覆盖）。
  - Phase 2 base_agent budget 分支：目标 ≥90%（U-P2.1 / U-P2.3 / U-P2.4 / U-P2.9 / U-P2.10 / U-P2.12）。
  - Phase 2 orchestrator except 分支：目标 100%（U-P2.2 / U-P2.5 / U-P2.6 / U-P2.13）。
- **不再做"总均必上不下"的纯数学推断**（v1 §6.2 末段已删）。
