# Test Plan v1 — eval-impl

> **作者**：verifier（team eval-impl）
> **日期**：2026-05-15
> **依据**：
> - `.multi-agent/eval-impl/plan/FINAL.md` v2（10 phase × GO 条件）
> - `.multi-agent/eval-impl/decisions/scope.md`（范围 = Phase 0-9 全量）
> - `.multi-agent/eval-impl/locks/approved-facts.md`（产物名 / env / 隔离 / `--ci` 边界）
> - `doc/evaluation/{procedure,acceptance}.md`（命令链 + schema + 阈值）
> **范围**：Phase 0-9 全量；Phase 7 (consistency) 与 Phase 6 (gate) 各自独立测；Phase 8 e2e 串通整链。
> **设计纪律**：
> - 三要素：输入 + 期望 + 断言锚点
> - 只测可观察行为（CLI 退出码 / stdout / 落盘文件 / pydantic 模型字段），不测私有方法签名
> - 失败:正常 ≥ 1:2
> - 不预测试 Tier-2 真实回放（scope.md §6 out-of-scope）

---

## 0a. 测试金字塔总览

| 层级 | 推荐占比 | 本方案实际 | 用例数 | 备注 |
|---|---|---|---|---|
| Unit | 60% | **62.9%** | 22（纯函数 / pydantic / 单 CLI 入口）| `test_common`, `test_schemas`, `test_ast_equiv`, `test_lock` 部分, `test_ground_truth`, `test_report_render`, `test_fork_name_check` |
| Integration | 25% | **28.6%** | 10（CLI 子模块跨文件交互，含 tmp_path + fake-bin / yaml / fixture）| `test_prepare`, `test_run`, `test_diff_against_golden`, `test_summarize`, `test_gate`, `test_consistency`, `test_lock`(--verify 路径) |
| E2E | 15% | **8.5%** | 3（procedure.md §2 整链 + CI yml meta）| `test_e2e_tier1`（2 用例）+ `test_ci_workflow_meta`（1 计入 e2e 因为它跨"代码 ↔ 工程基础设施"边界） |
| **合计** | 100% | 100% | **35** | |

偏离推荐 60/25/15 的理由：
- E2E 偏低（8.5% vs 15%）—— scope.md §6 明确 Tier-2/Tier-3 5 类后续工作；本期 Tier-1 仅 1 个 sample，e2e 串通即可，不堆量
- Integration 略高（28.6% vs 25%）—— CLI 入口测试天然跨文件（argparse + IO + json 落盘），无法纯 unit 化
- 整体偏向"集成型"是评估器项目的本性：核心价值在管线连通而非内部算法

执行预算（与 §15 合并口径）：单次 `pytest tests/eval/` ≈ 18s，CI 不增 > 30s。

### 12.2 mock 优先级

按团队规则：
- **优先**：`pytest-mock` 的 `mocker.patch(...)`
- 仅在 fixture 内部需要 context-manager 模式时用 `unittest.mock`

### 12.3 mypy strict 兼容

测试代码同样必须过 `mypy tests/eval`：
- fixture 加 `-> Generator[..., None, None]` 标注
- subprocess.run 返回值 cast 到 `subprocess.CompletedProcess[str]`
- dummy data 用 `cast(JsonDict, {...})` 避免类型推断歧义

### 12.4 覆盖率目标

`scripts/eval/` 模块整体 ≥ 80%，按 plan Phase 9 GO §2。每个 phase 的 unit 测试单独跑覆盖率 ≥ 80%。
